# Режими бота та промпти

## Огляд

Бот має 8 режимів роботи. Кожен використовує свій системний промпт та RAG-зону.

```
┌──────────────────────────────────────────┐
│              Головне меню                 │
├──────────┬───────────┬───────────────────┤
│ 💼 Coach │ 📚 KB     │ 🎓 LMS           │
│ 🔬 Cases │ 📋 Oper.  │ 📜 Certs         │
│ 🤝 Onboard│ 🔄 Combo │                  │
└──────────┴───────────┴───────────────────┘
```

---

## 1. Sales Coach (💼)

**Промпт:** `PROMPT_COACH` (480 рядків — найбільший)  
**RAG зона:** Products (k=12) або Products+Competitors (k=12+5)  
**Модель:** gpt-4o  
**Призначення:** Навчання менеджерів продавати препарати лікарям

**7 підрежимів (автовизначення):**
- НАВЧАННЯ — структурований розбір продукту (8 секцій)
- ТЕСТ — питання + оцінка + пояснення
- РОЛЬОВА ГРА — діалог менеджер↔лікар
- ЗАПЕРЕЧЕННЯ — SOS-формат або повний розбір
- ЕКСПРЕС-БРИФІНГ — скрипт на 30 секунд
- РОЗБІР ЗУСТРІЧІ — аналіз реальної зустрічі
- SOS — готова фраза під час дзвінка

**Приклад запиту:** "Лікар каже що Neuramis дорожчий за Juvederm, як відповісти?"

**Особливості промпту:**
- ТОП-6 правил на самому початку (антигалюцинація)
- 3 категорії продуктів (ін'єкційні / космецевтика / нутрієнти)
- Крос-сейл карта (волосся→IUSE HAIR, пігментація→Whitening)
- Заборона: **подвійних зірочок**, медичних призначень пацієнтам
- Пріоритет аргументів: фінанси лікаря > механізм > репутація > безпека > тривалість

## 2. База знань (📚)

**Промпт:** `PROMPT_KB`  
**RAG зона:** KB (k=15)  
**Модель:** gpt-4o-mini  
**Призначення:** Регламенти, HR, корпоративні правила

**Приклад запиту:** "Скільки днів відпустки мені належить?"

---

## 3. LMS / Навчання (🎓)

**Не використовує LLM** — працює через FSM + PostgreSQL  
**Структура:** Курси → Теми → Уроки (текст) → Тести (питання з варіантами)  
**Scoring:** passed/failed, score %, attempts  

**14 курсів:** Vitaran, Ellansé, Exoxe, Нейронокс, Petaran, Neuramis, Vitaran Skin Healer, IUSE Skin Booster, IUSE Collagen, IUSE Hair, HP Cell Vitaran + 3 додаткові

---

## 4. Розбір кейсів (🔬)

**Промпт:** `PROMPT_CASES`  
**RAG зона:** Products (k=12)  
**Модель:** gpt-4o-mini  
**Призначення:** Аналіз реальних діалогів з лікарями

**Приклад:** Менеджер вставляє текст зустрічі → бот аналізує що добре/погано, дає рекомендації

---

## 5. Операційні питання (📋)

**Промпт:** `PROMPT_OPERATIONAL`  
**RAG зона:** KB (k=15)  
**Модель:** gpt-4o-mini  
**Призначення:** Логістика, семінари, відрядження, SLA

**Приклад:** "Яка процедура організації семінару?"

---

## 6. Сертифікати (📜)

**Не використовує LLM і RAG** — SQL-пошук по sync_state  
**Призначення:** Пошук та завантаження PDF-сертифікатів за назвою бренду

**Приклад:** "Сертифікат Neuramis" → список PDF → завантаження

---

## 7. Комбо-протоколи (🔄)

**Промпт:** `PROMPT_COMBO`  
**RAG зона:** Products з фільтром category=combo (k=15)  
**Модель:** gpt-4o  
**Призначення:** Комбіновані протоколи (Ellanse + Vitaran, Petaran + Neuramis)

---

## 8. Онбординг (🤝)

**Не використовує LLM** — чеклист з PostgreSQL  
**Призначення:** 7-денний план для новачків (по днях)

---

## Маршрутизація запитів

```
Запит → detect_intent() [gpt-4o-mini] → mode_key
                    │
          ┌─────────┼──────────┐
          │         │          │
     "coach"    "kb"      "operational"
          │         │          │
     Продукт     Регламент   Логістика
     є?           │          │
     │            └──→ KB    └──→ KB
  Так  Ні
  │     │
  │   LLM route
  │     │
  ▼     ▼
Coach  KB/Other
```

**Оптимізація:** Якщо в запиті є назва продукту EMET — одразу Coach (без LLM routing, -300ms).
